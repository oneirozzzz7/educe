# Educe 前端设计宪法

> **一句话定位**：Educe 是你工作流的版本控制——它把你反复做的事变成可检视、可编辑、可携带的自动化资产。

---

## 骨架（永不变的信任承诺）

1. **Action 执行可见** — 每个操作卡片展示：做了什么、影响范围、可否撤销
2. **不可逆操作必须可恢复或确认** — 有快照/git 时用 undo 条，无法恢复时用确认
3. **Status 入口永远存在** — 一键查看系统对你的全部假设
4. **用户可关掉/修改任何学到的东西** — 每个反射、每个偏好都有 [编辑] [禁用]
5. **诚实降级** — 不确定时显式标注，不假装理解它不理解的
6. **反射永远署名** — 自动化运行时标注 ⚡，永不隐身，点击可编辑

## 肌肉优先级

| 优先级 | 肌肉 | 信号来源 | 速度 |
|--------|------|----------|------|
| **P0** | 回复详略度 (L0-L4) | 纠正=立即跳级；行为=累积3次±0.3 | 慢且准 |
| **P1** | 破坏性操作呈现 (C0-C4) | 降级=多次安全后慢降；升级=单次undo立即 | 不对称 |
| P2 | 进化状态展示量 | 用户点开过=记住展开 | 按需 |
| P3 | 领域上下文 | 后端 memory 处理 | 透明 |

## 哇时刻

> 不是"意外惊喜"，是**"我参与训练的东西真的起作用了，而且我能改它"**。

触发条件：系统从用户重复行为中提炼出反射候选，**主动提议固化**：
- 展示它学到了什么（可追溯：基于你最近 N 次操作）
- 提供 [接受] [看看它学到了什么] [不用]
- 接受后每次运行都署名 ⚡ + 可即时编辑

## 三个禁令

1. **永远不声明"我懂你了"** — 理解只通过行为改变体现，不通过文字声明
2. **永远不用悬念/营销话术** — 用事实陈述("这个模式再出现2次可固化")，不制造期待
3. **永远不让自适应突破安全地板** — 肌肉学到的偏好不能覆盖骨架的安全承诺

## 四条设计准则（判断器）

面对任何 UI 决策，问：

1. **骨架还是肌肉？** 关乎信任→硬编码；关乎偏好→从行为学
2. **体现还是声明？** 能体现就不声明
3. **猜错了用户纠正的代价多大？** 代价高→不猜，等显式信号
4. **信号可靠吗？** 纠正>重复>追问>速度(噪声)

## 前端核心交付物

- `educe status` — 当前协作快照（活跃反射 + 肌肉状态 + 待确认）
- `educe log` — 学习事件历史（每次升降级 + 证据）
- `educe diff` — 相对默认的偏离量
- `educe skills` — 反射库（可导出/导入/分享）
