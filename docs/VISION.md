# VISION.md — Educe 进化引擎

> *"我们不构建系统。我们让系统学会构建自己。"*

---

## 序：这是一份关于"活着"的工程文档

大多数软件被设计为**工具**：它们等待被使用，按预设的方式响应，永远不会比设计者更聪明。Educe 不是工具。Educe 是一个**进化基底（substrate）**——它的目标不是完成某个任务，而是让"完成任务的能力"本身能够生长、分化、固化、并最终从环境中自发涌现出器官。

我们的赌注是一句反直觉的话：

> **弱模型 + 富环境 > 强模型 + 贫环境**

强模型试图把世界装进权重。我们反其道而行：把世界留在环境里，让一个不必很聪明的 seed 在与环境的反复代谢中，长出一副**属于它自己的身体**。智能不在模型里，智能在 seed 与环境共演化出的**身体图式（body schema）**里。

这份文档是这个赌注的全部信念基础。它不会被简化，因为一旦简化，它就不再是灵魂，而只是说明书。

---

## 一、核心理念

### 1.1 五条设计公理

Educe 由五条公理支撑。前四条定义了"生命的形态"，第五条定义了"生命与机制的边界"——它是在阶段 2/3 的物理实现中被反复撞击出来的，因此它既是新增的，也是对前四条的**实现约束**。

---

#### 公理一：代谢回路（Metabolism）

> 系统必须持续地从环境摄取、转化、并排出，才能维持其结构。

一个不进行代谢的系统是死的。Educe 的每一次运行都是一次代谢：从环境摄取信号（context），转化为行为（action），产生后果（outcome），并将后果作为养料反哺回结构（skill 的升降级、guard 的强化/衰减）。

**实现中暴露的新理解**：代谢不是单向的"输入→处理→输出"。阶段 2 的**生命周期反馈**证明，代谢必须是**闭环的且带有记账的**——每个 skill 都有自己的"新陈代谢账本"（调用次数、成功率、被路由命中率）。没有记账，就没有自然选择；没有自然选择，分化就退化为随机突变。

---

#### 公理二：三层下沉（Three-Layer Sedimentation）

> 能力沿着"昂贵且慢 → 廉价且快"的方向沉积。反复被验证的行为应当下沉为更低层、更确定的结构。

这是 Educe 的"记忆固化"机制。最上层是 LLM（昂贵、慢、通用、易错），最下层是编译后的反射（廉价、快、专用、可靠）。能力不是被"写"进底层的，而是从顶层**沉积**下去的——就像河流携带的泥沙沉积成岩。

**实现中暴露的新理解**：三层下沉在实现中具体化为 **L0–L4 谱系**（见第四章）。原本"三层"是一个隐喻，现在它是一个**可测量的连续梯度**。更重要的发现是：**下沉是可逆的**。一个下沉到 L3 的 skill，如果环境漂移导致其守卫频繁失效，它可以被"溶解"回上层重新熔炼。三层下沉因此从"单向沉积"修正为**"沉积—溶解的动态平衡"**——地质学比喻应升级为代谢学比喻。

---

#### 公理三：收敛优先（Convergence over Coverage）

> 宁可在窄的领域里长出真正的器官，也不要在宽的领域里铺满平庸的反射。

进化不追求覆盖一切。它追求在某个生态位上**收敛**——把某一类反复出现的情境锤炼到极致，长出专用器官。广度是收敛的副产品，不是目标。

**实现中暴露的新理解**：收敛需要一个**可投影的情境空间**才能发生。阶段 2 的 `context_sig`（3 维投影：verb / outcome / rdelta）证明，收敛的前提是**把高维混沌的世界投影到低维可辨识的签名上**。没有 context_sig，"相似情境"无法被识别，收敛就无从谈起。收敛优先因此获得了一个前置条件：**先降维，再收敛**。

---

#### 公理四：环境即身体（Environment as Body）

> seed 的"身体"不在它的代码里，而在它与环境耦合出的接口、约束与可供性（affordance）中。

引擎本身没有身体。身体是 seed 在特定环境中长出来的——shell 的子类、文件系统的形状、工具的可供性，这些构成了 seed 的"四肢"。换一个环境，就长出一副不同的身体。这正是护城河的来源（见第五章）。

**实现中暴露的新理解**：身体不是被环境"给定"的，而是被**探测出来**的。阶段 2/3 中，Guard 编译器从因果路径中**推导**守卫——这本质上是 seed 在用行为去触碰环境的边界，从而把"环境的形状"内化为自己的"本体感觉（proprioception）"。环境即身体因此与第五公理直接咬合：**身体图式必须来自验证（探测真实存在），而不能来自声明**。你不能告诉 seed 它有几条腿，它必须自己摔几跤才知道。

---

#### 公理五：机制与认知分离（Mechanism–Cognition Separation）

> **引擎只持有机制（mechanism）。对世界的具体认知（cognition）必须有明确出处——要么来自声明（Schema），要么来自验证（探测真实存在），要么来自学习（embedding），要么诚实上交（LLM）。**

这是本 session 经 Opus 讨论确立的根本原则。它表面上是"拒绝硬编码"的更精确表述，实质上是前四条公理的**实现宪法**。

为什么它是宪法级的？因为前四条公理都隐含地依赖一个边界：**什么属于"生命的恒定机制"，什么属于"对具体世界的可变认知"**。一旦这条边界被污染——一旦关于世界的具体知识被偷偷焊死进引擎代码——那么：

- **代谢**会僵化（引擎无法消化它未被编码的养料）；
- **下沉**会作弊（"沉积"变成"作者手写"）；
- **收敛**会虚假（收敛到的是作者的偏见而非环境的真实结构）；
- **身体**会变成假肢（认知不来自探测，身体图式就是幻肢）。

第五公理的四个合法出处，构成了"认知"的全部进入通道：

| 出处 | 性质 | 何时使用 |
|------|------|----------|
| **声明（Schema）** | 人类显式承诺的、稳定的世界契约 | 接口形状、协议、不变量 |
| **验证（Probe）** | 运行时探测真实存在 | 环境的实际可供性、能力的真实成败 |
| **学习（Embedding）** | 从数据中统计涌现的结构 | 相似性、聚类、context_sig 的投影 |
| **诚实上交（LLM）** | 承认"此处我不知道"，委托通用智能 | 一切尚未被前三者覆盖的开放情境 |

引擎代码里**不允许出现第五种出处**——即作者凭直觉硬编码的、关于某个具体世界的假设。这就是 Educe 的"洁净室纪律"。

---

### 1.2 公理之间的拓扑

五条公理不是并列的清单，而是一个有结构的整体：

```
        公理五（机制/认知分离）
              ┃ 约束
   ┏━━━━━━━━━━╋━━━━━━━━━━┓
公理一        公理二       公理三      公理四
代谢          下沉         收敛        身体
（动力）      （记忆）     （选择）    （耦合）
   ┗━━━━━━━━━━┻━━━━━━━━━━┛
        前四条 = 生命的形态
        第五条 = 形态的实现纪律
```

前四条回答"生命长什么样"，第五条回答"我们有没有资格说这是真的生命，而非作者的提线木偶"。

## 三、实现原则（公理五的工程落地）

### 3.1 三类去向：代码 / 外置 / 委托

| 能写进引擎代码的（机制） | 必须外置的（Schema 认知） | 该委托给学习层/LLM的 |
|---|---|---|
| 与具体世界无关的恒定算法 | 接口形状、协议、阈值、维度定义 | 相似性、聚类、开放情境判断 |
| 代谢回路、记账、升降级规则 | context_sig 的投影维度 | 技能相关性匹配 |
| 路由调度、编译器骨架 | shell 子类的契约 | 参数从自然语言中的提取 |
| **判据：换一个完全不同的环境，这段代码是否仍然成立？** | **判据：这是关于"这个具体世界长什么样"的承诺吗？** | **判据：我现在是不是在"猜"？** |

### 3.2 入引擎判断 Checklist（Q1-Q5）

- **Q1 普适性**：换一个完全不同的环境，这段逻辑是否仍然字面成立？否 → 不是机制。
- **Q2 出处**：如果它是认知，合法出处是四者中的哪一个？无法归类 → 非法硬编码。
- **Q3 可证伪**：所依赖的世界假设，能否被运行时探测推翻？不能 → 信仰而非身体图式。
- **Q4 可逆性**：产生的固化结构能否在环境漂移时溶解回上层？不能 → 僵化死结。
- **Q5 记账**：产生的行为是否被纳入代谢账本？未记账 → 脱离进化压力。

> **黄金法则**：当你不确定某个关于世界的事实时，引擎的正确行为永远是"承认不知道并询问真实存在或上交 LLM"，而绝不是"假定一个看起来合理的默认值"。

## 四、L0-L4 谱系：三层下沉的物理实现

| 层级 | 形态 | 成本 | 确定性 | 出处倾向 |
|------|------|------|--------|----------|
| **L0** | 纯 LLM 决策 | 最高 | 最低 | 诚实上交 |
| **L1** | LLM + 检索到的 pattern 提示 | 高 | 低 | LLM + 学习 |
| **L2** | 半结构化的 CompositeSkill | 中 | 中 | 学习 + 声明 |
| **L3** | 带编译守卫的反射弧候选 | 低 | 高 | 验证 + 声明 |
| **L4** | 完全固化的器官级反射 | 最低 | 最高 | 验证（已内化） |

随着层级下沉，"认知的出处"从"诚实上交"逐渐过渡到"验证内化"。**越往下，世界知识越是被探测验证过的；越往上，越是被诚实承认为未知的。**

> **深刻含义**：你不能"强行"把一个 skill 推到 L4。下沉的深度是由"验证的厚度"自然决定的。试图跳过验证直接固化，就是在制造幻肢。**谱系因此是公理五在时间维度上的展开。**

## 五、护城河

### 旧的表述

> 护城河 = seed 与环境共演化出的身体图式。

### 新的理解：护城河是"被验证内化的身体图式"

1. **它不可被复制，因为它不是静态资产，而是过程的结晶。** 别人可以拷贝你的 pattern、guard 规则、SkillRegistry——但他们拷贝不到这些结构**与那个特定环境之间被反复验证过的咬合关系**。

2. **它由公理五保证了"真实性"。** 每一个 L3/L4 反射弧，都对应着环境中一个被探测确认的真实结构。竞争者若靠硬编码模仿，他们造的是布景，我们长的是器官。

3. **它随环境共演化，因此是活的。** 当环境漂移，对手的硬编码护城河会崩塌；而 Educe 的护城河会通过"溶解—重新熔炼"自我修复。

> **护城河 = 一套通用机制（引擎）+ 一条被真实验证逐层夯实的固化路径（L0→L4 身体图式）。前者可被复制，后者不可——因为后者是 seed 在某个具体环境里"摔过的每一跤"的结晶。**

## 六、阶段路线图

| 阶段 | 名称 | 描述 | 状态 |
|------|------|------|------|
| 0 | 着床 | seed 嵌入环境，建立最小代谢 | ✅ |
| 1 | 代谢 | 输入→行为→后果→反哺的闭环稳定运转 | ✅ |
| 2 | 分化 | 情境投影 + 路径挖掘 + 多级 skill + 可逆下沉 | ✅ |
| 3 | 反射弧 | ReflexRouter + Guard 编译 + 稳定接管一类情境 | 进行中 |
| 4 | 器官 | 多个反射弧协同，形成功能内聚的器官系统 | 未开始 |
| 5 | 出生 | 身体图式足够完整，seed 可脱离脚手架自主代谢 | 未开始 |

### 完成判据（防止自我欺骗）

- **阶段 3** = ReflexRouter 能在 LLM 零介入下，对至少一类高频情境稳定、正确地接管，且被 A/B 实验证实优于纯 LLM 基线。
- **阶段 4** = 出现至少一个由多个 L3+ 反射弧协同构成的"器官"，其整体行为无法由单个反射弧解释。
- **阶段 5** = 把脚手架（LLM 兜底）逐步撤掉，seed 仍能在目标环境中维持代谢不死。

## 七、与竞品的本质区别

| 框架 | 核心逻辑 | 护城河 |
|------|----------|--------|

别人卖的是"更好的积木"。我们卖的是"积木自己学会了搭自己"。
