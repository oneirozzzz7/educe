# Educe 核心愿景与架构路线图

## 我们是谁

> **Educe 不做"连接万物的 Agent"。我们做"让系统活过来"的进化引擎。**
>
> Seed 栖息在环境里，把环境长成自己的身体，把试错蒸馏成世界模型，让任何系统从被操作的机器变成自我进化的有机体。连接是它活着的副产品，不是目标。

## 核心理念

### 为什么是弱模型

强模型靠庞大先验"一步到位"。Educe 用弱模型的**"现实反馈快速迭代"**达到同样结果。

弱模型 + 富环境 > 强模型 + 贫环境。

### 四个设计公理

1. **代谢回路**：试探 → 观察后果（含长期）→ 固化或淘汰 → 改变自身结构。这是一切进化的物理前提。
2. **三层下沉**：新交互进入意图层（慢推理）→ 重复稳定后下沉为技能层（自动化）→ 高频固化为反射层（零成本）。今天所有 Agent 把一切塞进意图层——Educe 让交互自动下沉。
3. **收敛优先**：弱模型每个决策点只面对"刚好能驾驭"的环境切面。不是广度优先（工具越多越好），是收敛优先（少而准）。
4. **环境即身体**：Seed 与环境的关系不是"主体操作客体"，而是"器官与身体"。Seed 用熟的工具不再是外部调用，而是长进了它的身体图式。

### 护城河

一个在客户环境里栖息三个月的 seed，它的身体图式已和那个环境长在一起。换"更强的通用模型"进来也得从零长起。**护城河不在模型，在 seed 与环境共同演化出的那层"身体"。**

### 连接万物的终点

> 连接万物的终点，是越来越少地需要真实连接。

就像高手下棋不用走出来才知道好坏。连接是手段，**内化成世界模型才是目的**。

## 架构路线图

### 阶段 0：着床 — 让后果可观测

**能力跃迁**：从"seed 做了什么"到"seed 知道自己做的事产生了什么后果"

**核心交付**：
- 因果账本（Consequence Ledger）：每个决策点记录 `(context, action, outcome)` 三元组
- Outcome Hook：行为循环每个 action 执行后强制写账本
- 即时奖励函数：规则打分（成功/失败/用户确认/拒绝）
- 前端后果可视化

**复用**：行为循环、知识系统存储、ConnectorRegistry、WebSocket  
**新建**：因果账本 schema + OutcomeCapturer + 前端后果徽章

**验证**：100 个真实任务，95%+ 决策点产出结构化三元组。能回答"过去 7 天哪类 action 失败率最高"。

**独立价值**：哪怕后面全停，也得到"行为-后果可观测系统"，debug 和信任巨大提升。

---

### 阶段 1：代谢启动 — 让后果改变下次行为

**能力跃迁**：从"记录后果"到"因为上次失败，这次换个走法"

**核心交付**：
- 延迟后果回填（Outcome Reconciler）：关联 5 分钟/1 天后的延迟信号
- 决策前因果检索：查"相似 context 下这个 action 历史 outcome 如何"，注入 prompt
- 激发引擎重构：fitness 函数从开环换成**账本驱动的真实后果累积奖励**

**重构**：激发引擎核心（fitness = Σ ledger rewards）  
**新建**：Outcome Reconciler、因果检索

**验证**（最关键一关）：
- 同任务 A/B 测试：因果注入组 vs 对照组，35B 失败率相对下降 ≥ 20%
- 闭环优于开环证明：把 fitness 换回开环，演化收益应显著退化

> ⚠️ 这一关过不了就不往下走。它验证"代谢回路"是否真实成立。

---

### 阶段 2：分化 — 重复路径下沉为技能

**能力跃迁**：从"每次都推理"到"这条路我走熟了，打包成一个动作"

**核心交付**：
- 路径挖掘器：在账本中挖高频 + 高奖励 + 低方差的 decision-point 序列
- 技能固化：编译为 composite capability，注册进 ConnectorRegistry
- 弱模型在意图层面对的分支自动变少（收敛优先的物理实现）

**验证**：系统自动提出 ≥ 5 个 composite skill，调用时决策步数下降 ≥ 40%，token 消耗下降。

---

### 阶段 3：反射弧 — 技能内化为反射

**能力跃迁**：从"调用技能要先想"到"看到 pattern 直接做，跳过 LLM"

**核心交付**：
- 超高频 + 超稳定技能 → 训练轻量分类器/规则，绕过决策树 prompt
- LLM 退居"反射失败时的兜底"和"新情况的探索者"

**验证**：高频路径 ≥ 30% 决策不经过 LLM，延迟下降，准确率不降。

---

### 阶段 4：器官形成 — 不可迁移的环境结构

**能力跃迁**：从"通用 agent"到"这个 seed 离开这个环境就废了，但在这里无可替代"

沉淀的技能/反射/因果账本，显式结构化为**环境特定的器官**。护城河显性化。

---

### 阶段 5：出生 — seed 衍生子 seed

**能力跃迁**：成熟 seed 能 fork 出子 seed，携带遗传（积累的技能/反射/偏好），在新环境中变异适应。

## 最弱模型优先的验证哲学

每个阶段的验证必须在 Qwen3.6-35B-A3B 上通过。如果弱模型做不到，说明我们的"框架激发"不够——要么框架该承担更多，要么环境反馈不够密。强模型能做到不算数。

---

## 与竞品的本质区别

| 框架 | 核心逻辑 | 护城河 |
|------|----------|--------|
| LangChain | 工具编排（组装管道） | 生态/工具数量 |
| AutoGen/CrewAI | 多 Agent 协作 | 编排复杂度 |
| **Educe** | **代谢进化（seed × 环境共生）** | **seed 与环境的共演化身体** |

别人卖的是"更好的积木"。我们卖的是"积木自己学会了搭自己"。
